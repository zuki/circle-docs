DMAバッファの要件
-----------------------

CircleにはDMAとCPUデータキャッシュを使用して動作を高速化しているデバイスドライバが
あります。このようなシステムでデータコヒーレンシを管理するにはデバイスドライバには
特別なサポートが必要です。このようだデバイスドライバは、DMA転送の開始前にデータ
バッファがデータキャッシュからSDRAMに書き出されること、DMA転送の完了後にデータ
バッファがデータキャッシュにロードされることを保証しなければなりません。

これはシステム制御オペレーションを使用して実行されますが、DMA転送に使用される
データバッファはシステムのデータキャッシュラインのサイズ（Raspberry Pi 1とZeroでは
32バイト、Raspberry Pi 2-4では64バイト）にアライメントされている必要があります。
アライメントはバッファのベースアドレスとサイズに適用する必要があります。この要件が
満たされていない場合、Circleアプリケーションは動作しますが、特定の条件下では
アライメントされていないDMAバッファの近くに保存されたデータが破損する可能性があります。
このような問題の検出は難しいでしょう。

そのため、デバイスドライバでDMA用に使用されるデータバッファをキャッシュアラインする
ことが重要です。性能上の理由によりCircleの一部のデバイスドライバはアプリケーションから
デバイスドライバに渡されたデータバッファをDMAオペレーションに直接使用するため、この要件は
アプリケーションバッファにも対象となります。このファイルでは、アプリケーションで
どのような状況でこの要件が必要なのか、またキャッシュアラインされたデータバッファを
どのように定義できるのかを説明します。

DMAバッファを定義する場所
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

DMAオペレーションに使用されるデータバッファはキャッシュアラインメントされたDMA
バッファとして定義する必要があります。これは多くの場合、デバイスドライバに
適用されますが、デバイスドライバを書くことはめったにありません。パフォーマンス上の
理由から、以下の条件下ではCircleアプリケーションにも適用されることがあります。

* CDMAChannelメソッドに渡されるバッファはキャッシュアラインメントされたDMAバッファで
  なければなりません。

* CSPIMasterDMAメソッドに渡されるバッファはキャッシュアラインメントされたDMAバッファで
  なければなりません。

* 低レベルUSB転送のためにUSBホストコントローラに直接アクセスする場合、次のメソッドに
  渡されるバッファはキャッシュアラインメントされたDMAバッファでなければなりません。 ControlMessage、GetDescriptor、Transfer、SubmitBlockingRequest、SubmitAsyncRequest。

* ネットワークデバイスでフレームを直接送受信する独自のネットワークスタックを使用する
  場合、そのネットワークデバイスに渡されるバッファはキャッシュアラインメントされた
  DMAバッファでなければなりません。ただし、これはCSocketクラスに渡されるバッファには
  適用されません。

* CUSBBulkOnlyMassStorageDeviceメソッドに渡されるバッファはパフォーマンス上の理由で
  キャッシュアラインメントされたDMAバッファであるべきです。キャッシュアラインメント
  されていない場合、ドライバはそれを検出し、キャッシュアラインメントされたDMAバッファを
  独自に提供することになります。これには memcpy() 操作が必要となり、パフォーマンスが
  低下します。SDカードデバイスドライバ CEMMCDevice はDMAを使用しないため、キャッシュ
  アラインメントされたDMAバッファは必要ありません。

DMAバッファの定義
^^^^^^^^^^^^^^^^^^^^^

DMA転送に使用されるデータバッファは次のようにさまざまな場所に格納することができます。

* スタック,
* ヒープ（"new"演算子またはmalloc()を使用）,
* 静的データセグメント (.data, .bss).

1. スタック上でキャッシュアラインメントされたDMAバッファを定義する推奨の方法は以下の
   通りです。

   .. code-block:: C

      #include <circle/synchronize.h>

      some_function()
      {
        ...
        DMA_BUFFER (unsigned char, MyBuffer, 100);
        ...

   DMA_BUFFER() マクロはRaspberry Pi 2-4 (キャッシュラインサイズ64バイト)で以下の
   配列定義を生成します。

   .. code-block:: C

	      unsigned char MyBuffer[128] __attribute__ ((aligned (64)));

   これで定義された配列には "sizeof" 演算子を使ってはいけないことに注意してください。
   配列にはキャッシュアライメントのためのパディングバイトがある可能性があるからです。

2. ヒープから割り当てたデータバッファは常にキャッシュアラインされています。

   .. code-block:: C

	      unsigned char *pDMABuffer = new unsigned char[100];

   pDMABufferはDMA用に直接使うことができます。"new"演算子を使ってC++オブジェクトを
   作成する場合、そのクラス定義にキャッシュアラインされたデータバッファメンバ変数を
   含めることができます。

   .. code-block:: C

        class CSomeClass
        {
          ...
          DMA_BUFFER (unsigned char, m_MyBuffer, 100);
          ...

3. DMAバッファは関数の外側で静的に定義したり、上に示したようにDMA_BUFFER()マクロを
   使って関数やクラス定義の内側で "static" キーワードを追加して定義したりすることが
   できます。
